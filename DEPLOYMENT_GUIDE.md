# دليل نشر وتوزيع النظام: واجهة أمامية (Vercel) + خلفية (Render)

يرشدك هذا الدليل خطوة بخطوة لكيفية فصل ونشر تطبيق بوابة الاستعلام والشهادات ودفع المصاريف على السحابة:
1. **الواجهة الأمامية (Frontend):** يتم نشرها على منصة **Vercel**.
2. **الخلفية الرقمية وقاعدة البيانات (Backend):** يتم نشرها على منصة **Render**.

---

## 🏛️ الهيكل المعماري بعد الفصل

*   **الواجهة الأمامية (React + Vite):**
    *   ترسل الطلبات إلى عنوان السيرفر على Render عبر المتغير `VITE_API_BASE_URL`.
    *   تحتوي على ملف التوجيه الآمن `vercel.json` لتفادي مشاكل إعادة تحميل الصفحة (404 Error) عند استعراض روابط التحقق التلقائي (QR Code).
*   **الخلفية الرقمية (Express + Node + TS):**
    *   تدعم الـ CORS بشكل كامل للسماح للواجهة الأمامية على Vercel بالاتصال بها بأمان.
    *   تدير خيارات بوابة كاشير (Kashier Payment Gateway) وتدعم البيئة التجريبية (Sandbox) والبيئة الحية (Production).

---

## 🚀 الجزء الأول: نشر الخلفية (Backend) على Render

سيرفر Express يعتمد على ملف `server.ts` ومجلد `server/` لإدارة قاعدة البيانات المحلية وتوقيع المعاملات لـ كاشير.

### 1. خطوات النشر على Render:
1. قم بإنشاء حساب مجاني على [Render.com](https://render.com).
2. اضغط على **New +** ثم اختر **Web Service**.
3. قم بربط مستودع الـ GitHub الخاص بك بـ Render.
4. قم بضبط الإعدادات التالية في لوحة تحكم Render:
    *   **Name:** `certified-students-backend` (أو أي اسم تفضله)
    *   **Region:** اختر الأقرب للمستخدمين (مثال: `Frankfurt (EU)`)
    *   **Language:** `Node`
    *   **Branch:** `main`
    *   **Build Command:** `npm install && npm run build`
    *   **Start Command:** `npm run start` (والذي سيقوم بتشغيل السيرفر المجمع في `dist/server.cjs`)

### 2. متغيرات البيئة على Render (Environment Variables):
اضغط على قسم **Advanced** ثم **Add Environment Variable** وأضف الآتي:

| اسم المتغير | القيمة | الوصف |
| :--- | :--- | :--- |
| `NODE_ENV` | `production` | لتشغيل البيئة بخصائص الإنتاج والسداد الحقيقي |
| `GEMINI_API_KEY` | *(اختياري)* | إذا كنت تستخدم ميزات الذكاء الاصطناعي مستقبلاً |

*(ملاحظة: السيرفر يدعم الحفظ التلقائي في ملف `database.json`. إذا كنت تستخدم الخطة المجانية على Render، سيتم إعادة ضبط الملف عند إعادة تشغيل السيرفر. لحل ذلك وحفظ البيانات للأبد، يمكنك استخدام نظام قواعد البيانات السحابية Firebase Firestore، أو ببساطة استخدام ميزة **Disk / Persistent Volume** المجانية على Render لربط مسار حفظ البيانات).*

## 🎨 الجزء الثاني: نشر الواجهة الأمامية (Frontend) على Vercel

الواجهة الأمامية مبنية باستخدام React و Vite وسيتم نشرها كموقع استاتيكي سريع جداً ومحمي على شبكة CDN الخاصة بـ Vercel.

### 1. خطوات النشر على Vercel:
1. قم بإنشاء حساب مجاني على [Vercel.com](https://vercel.com).
2. اضغط على **Add New...** ثم اختر **Project**.
3. قم باستيراد مستودع الـ GitHub الخاص بك.
4. في لوحة ضبط Vercel، قم بتوسيع قسم **Environment Variables** وأضف المتغير التالي:

| اسم المتغير | القيمة | الوصف |
| :--- | :--- | :--- |
| `VITE_API_BASE_URL` | `https://certified-students-backend.onrender.com` | رابط سيرفر الخلفية الخاص بك الذي حصلت عليه من Render (بدون شرطة مائلة في النهاية `/`) |

5. اضغط على **Deploy**. سيقوم Vercel تلقائياً بقراءة `vercel.json` وبناء المشروع بنجاح.

## 💳 الجزء الثالث: تفعيل واختبار بوابة كاشير (Kashier Sandbox)

بناءً على طلبك، تم إعداد النظام تلقائياً ليعمل بالرابط والوضع التجريبي لـ كاشير (**Kashier Sandbox**) افتراضياً:

1. **الوضع التجريبي الافتراضي:**
   * تم ضبط القيمة الافتراضية لـ `mode` في قاعدة البيانات لتكون `'sandbox'` بدلاً من المحاكي الداخلي.
2. **رابط الدفع التجريبي:**
   * عندما يقوم الطالب بالضغط على دفع المصاريف للشهادة، سيتم تحويله تلقائياً لصفحة الدفع الآمنة الرسمية الخاصة بـ كاشير بيئة الاختبار:
     `https://checkout.kashier.usr.sh/`
3. **مفاتيح الحساب التجريبي الخاصة بك:**
   * يمكنك تسجيل الدخول للوحة تحكم الإدارة الآمنة للموقع عبر الرابط: `/admin` (اسم المستخدم: `admin` وكلمة المرور: `password123`).
   * اذهب لعلامة تبويب **إعدادات كاشير** لتغيير الـ Merchant ID، الـ API Key، والـ API Secret التجريبي أو الحقيقي بأي وقت وحفظه سحابياً على الفور.

## 📂 خيار: فصل المشروعين في مستودعين منفصلين (مستحب للتنظيم)

إذا كنت ترغب بإنشاء مستودعين منفصلين تماماً على GitHub:

### 1. مستودع الواجهة الأمامية (Frontend Project):
يحتوي فقط على ملفات العرض والتصميم:
*   مجلد `src/` (بما فيه `src/config.ts`)
*   ملف `index.html`
*   ملف `vite.config.ts`
*   ملف `tsconfig.json`
*   ملف `package.json` (يمكنك حذف تبعيات express و esbuild و tsx منه لتخفيف الحجم)
*   ملف `vercel.json`

### 2. مستودع الخلفية الرقمية (Backend Project):
يحتوي فقط على السيرفر وقاعدة البيانات:
*   ملف `server.ts`
*   مجلد `server/` (الذي يحتوي على ملف `db.ts`)
*   ملف `tsconfig.json`
*   ملف `package.json` (يمكنك إبقاء تبعيات express و dotenv و esbuild و tsx وحذف تبعيات react)
